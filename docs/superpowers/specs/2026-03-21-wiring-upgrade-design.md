# MurmuraScope Wiring Upgrade — Design Spec

**Date:** 2026-03-21
**Status:** Approved
**Scope:** Connect all disconnected infrastructure, replace Tier system with stochastic activation, implement true Bayesian belief updates

---

## Context

A Claims vs Reality audit identified 9 wiring gaps where working infrastructure was built but never connected to the simulation loop. This upgrade connects all gaps and replaces the Tier 1/2 architecture with a MiroFish-inspired stochastic activation model.

### Key Decisions Made

1. **Priority:** Wiring (Category A) first, validation (Category B) later
2. **Execution order:** Follow simulation pipeline (Group 1→2→3), not impact ranking
3. **Test strategy:** Implement all changes first, then batch-fix broken tests + add new integration tests
4. **Tier architecture:** Remove Tier 1/2 entirely; adopt stochastic activation where all activated agents use LLM
5. **Bayesian belief:** Implement true Bayes formula (not just rename)
6. **Rationale for full LLM:** Background agents with LLM can produce unexpected emergent behavior that rule-based agents cannot — this is the engine's core value proposition

## Phase 0 — Tier Removal + Stochastic Activation

### Current State

```
simulation_runner.py:
  → _assign_tiers(): SELECT TOP 30 by influence → tier=1, rest tier=2
  → Group 2: if tier==1 → LLM deliberation; if tier==2 → skip or lite_hooks
  → All kg_driven hooks guard: if tier==1 only
```

### New Architecture

**Stochastic Activation Model:**

1. **Agent Profile** gains 3 behavioral params (generated by KGAgentFactory via LLM, or inferred from demographics in HK mode):
   - `activity_level: float [0,1]` — per-round activation probability
   - `influence_weight: float [0,3]` — post visibility weight in feed ranking
   - `response_delay: int [0,5]` — reaction delay in rounds (future use)

2. **Per-round activation** via `get_active_agents_for_round()`:
   - Each agent independently: `random() < activity_level`
   - All active agents → LLM deliberation (no tier distinction)
   - Inactive agents → silent this round (retain previous state, no lite_hooks)

3. **Model routing** (cost control):
   - `is_stakeholder=True` agents → `AGENT_LLM_MODEL` (stronger model)
   - `is_stakeholder=False` agents → `AGENT_LLM_MODEL_LITE` (flash-lite)
   - `is_stakeholder` marked by ScenarioGenerator at Step 2
   - Both paths are real LLM — quality differs, not mechanism

4. **Semaphore:** `max_concurrent = min(50, active_count)`

5. **lite_hooks.py** new scope:
   - No longer used in main simulation loop
   - Serves SwarmEnsemble Phase B replicas only

6. **Stakeholder floor:** `is_stakeholder=True` → `activity_level = max(activity_level, 0.8)`

7. **Reproducibility:** New `activation_seed` parameter; fixed seed → deterministic activation sequence

### HK Mode activity_level Inference Rules

```
Age 18-35:  0.7-0.9
Age 36-55:  0.4-0.6
Age 55+:    0.2-0.4
High income: influence_weight × 1.3
Civil servant/teacher/journalist: activity_level + 0.1
```

### Cost Estimate (STANDARD: 300 agents, 20 rounds)

```
Average activity_level ≈ 0.5 → ~150 active agents/round
Stakeholders ~30 → gemini-flash ($0.002/call)
Background ~120 → gemini-flash-lite ($0.0003/call)
Per-round cost: 30×0.002 + 120×0.0003 = $0.096
20 rounds: $1.92 (vs current ~$1.89 — nearly identical)
```

## Phase 1 — Upstream Wiring (Group 1 / Pre-Group-2)

### ① Episodic Memory → All Agent Decisions

**Gap:** Only kg_driven Tier 1 agents retrieve memory during deliberation.
**Fix:** Every active agent retrieves top-5 relevant memories before LLM deliberation.

```
Flow:
  Group 2 decision step:
    memories = await agent_memory.get_relevant_memories(agent_id, context, top_k=5)
    → inject into LLM prompt as {recent_memories}
```

**Files:** `decision_prompts.py` (add `{recent_memories}` placeholder), `simulation_runner.py` (call `get_relevant_memories()` before decision)

### ② Emotional Contagion → kg_driven Mode

**Gap:** `_process_emotional_contagion()` only fires in hk_demographic mode.
**Fix:** Remove mode guard; both modes share the same emotional contagion path.

**Files:** `simulation_hooks_social.py` (remove hk_demographic check), `simulation_runner.py` (unify Group 1 emotional hook)

### ③ Cognitive Fingerprint → All Modes

**Gap:** Only kg_driven mode generates and uses cognitive fingerprints.
**Fix:** HK mode infers fingerprint from demographic profile via rule-based mapping.

```
Mapping:
  political_stance → authority/loyalty values
  age + education → openness/conformity
  income → security/prestige
```

**Files:** `agent_factory.py` (add `_infer_fingerprint_from_demographics()`), `decision_prompts.py` (add fingerprint context for both modes)

## Phase 2 — Core Wiring (Group 2)

### ④ Feed Ranking → Agent Cognition Loop

**Gap:** `agent_feeds` table populated but agents never read their feed during decisions.
**Fix:** Each active agent retrieves top-5 feed items as LLM decision context.

```
Flow:
  Group 2, before decision:
    feed_items = await feed_ranker.get_agent_feed(agent_id, round, limit=5)
    → inject into LLM prompt as {feed_context}
    "你最近睇到嘅帖文：\n{feed_items_formatted}"
```

**Anti-echo-chamber:** Feed ranking adds diversity injection — at least 1/5 items from opposing stance.

**Files:** `simulation_runner.py`, `decision_prompts.py` (add `{feed_context}`), `feed_ranker.py` (add `get_agent_feed()` query helper)

### ⑤ Bayesian Belief Update (True Implementation)

**Gap:** Current system uses linear weighting (Anderson Integration Theory), mislabeled as "Bayesian."
**Fix:** Implement true Bayes formula.

```
New formula:
  prior = agent.belief[topic]                         # P(H)
  lr = compute_likelihood_ratio(event, agent)         # P(E|H) / P(E|¬H)
    - lr derived from: event.impact × credibility × (1 ± confirmation_bias)
    - lr > 1: evidence supports belief direction
    - lr < 1: evidence contradicts
  prior_odds = prior / (1 - prior)
  posterior_odds = prior_odds × lr
  posterior = posterior_odds / (1 + posterior_odds)    # back to probability
  clamp to [0.02, 0.98]                              # prevent certainty lock
```

**Files:** `belief_system.py` (new `bayesian_update()` replacing `linear_update()`; new `compute_likelihood_ratio()`), `belief_propagation.py` (cascade uses `bayesian_update`)

### ⑥ Debate Delta → Belief Writeback

**Gap:** ConsensusDebateEngine computes belief_delta but only persists to DB, never applies to agent state.
**Fix:** Apply debate deltas immediately after debate round.

```
Flow:
  After _kg_consensus_debate():
    for result in debate_results:
      await belief_system.apply_debate_delta(
        agent_id=result.agent_id,
        topic=result.topic,
        delta=result.belief_delta    # already clamped ±0.15/exchange, ±0.20/round
      )
  apply_debate_delta converts delta → evidence strength → likelihood ratio → bayesian_update
```

**Files:** `simulation_runner.py` (wire writeback after debate), `belief_system.py` (add `apply_debate_delta()`)

## Phase 3 — Downstream Wiring (Group 3 + Branching)

### ⑦ Relationship → Behavior Feedback

**Gap:** Gottman scores computed but don't trigger dissolution; trust doesn't influence decisions.

**Fix A — Gottman triggers dissolution:**
```
relationship_lifecycle.py: DISSOLVED condition becomes:
  if gottman_ratio > 0.8 OR (commitment < 0.1 AND trust < -0.3)
```

**Fix B — Trust influences decisions:**
```
decision_prompts.py: inject top-3 trusted + top-3 distrusted agents
  "你最信任嘅人：{trusted_names}（立場：{stances}）"
  "你最唔信任嘅人：{distrusted_names}"
```

**Fix C — Relationship crisis affects emotion:**
```
emotional_engine.py: relationship crisis event → valence -0.1, arousal +0.15
```

**Files:** `relationship_lifecycle.py`, `decision_prompts.py`, `emotional_engine.py`, `simulation_runner.py`

### ⑧ Shock → MacroController

**Gap:** Shock injection creates a post only; does not directly modify macro parameters.
**Fix:** Dual-path shock — narrative (existing) + macro (new).

```
ScheduledShock model: add macro_effects: Optional[dict[str, float]]
  e.g. {"interest_rate": +0.25, "consumer_confidence": -5}

If macro_effects present → call MacroController.apply_shock()
If absent → narrative-only (backward compatible)
```

**Files:** `simulation.py` (shock endpoint), `macro_controller.py` (wire `apply_shock()`), ScheduledShock model (add `macro_effects`)

### ⑨ Branch Deep Copy → Full State

**Gap:** Only copies agent_profiles + memories + actions (~50% of state).
**Fix:** Copy all dynamic state tables.

```
Additional tables to copy:
  - belief_states (topic-stance pairs at fork_round)
  - emotional_states (VAD at fork_round)
  - agent_relationships (trust + relationship type)
  - kg_edges (knowledge graph edges)
  - cognitive_dissonance (cognitive state records)

Use INSERT INTO ... SELECT for batch efficiency.
```

**Files:** `simulation_branches.py` (expand `_copy_session_state()`), `swarm_ensemble.py` (add `agent_relationships` + `kg_edges` + `cognitive_dissonance` to existing copy)

## Risk Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Stochastic activation increases run variance | Hard to reproduce bugs | `activation_seed` parameter; fixed seed → deterministic activation |
| Full LLM increases per-round latency | STANDARD ~30s/round → ~60s/round | Semaphore + batch requests + async gather |
| Bayesian update unstable near extremes (0.01/0.99) | NaN or belief lock | Clamp [0.02, 0.98]; unit tests cover edge cases |
| Feed→cognition creates echo chamber positive feedback | Beliefs converge too fast | Feed diversity injection: ≥1/5 posts from opposing stance |
| Branch deep copy increases DB I/O | Branch creation slower | `INSERT INTO ... SELECT` batch copy |
| Removing tier guards breaks unknown dependencies | Unexpected test failures | Run full test suite after Phase 0 to establish baseline |
| Activity_level inference inaccurate | Key agents silenced | Stakeholder `activity_level` floor = 0.8 |
