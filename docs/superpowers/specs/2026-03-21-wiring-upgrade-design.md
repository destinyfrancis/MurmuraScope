# MurmuraScope Wiring Upgrade — Design Spec

**Date:** 2026-03-21
**Status:** Approved (rev 2 — post spec review)
**Scope:** Connect all disconnected infrastructure, replace Tier system with stochastic activation, implement true Bayesian belief updates

---

## Context

A Claims vs Reality audit identified wiring gaps where working infrastructure was built but never connected to the simulation loop. This upgrade connects all gaps and replaces the Tier 1/2 architecture with a MiroFish-inspired stochastic activation model.

### Key Decisions Made

1. **Priority:** Wiring (Category A) first, validation (Category B) later
2. **Execution order:** Follow simulation pipeline (Group 1→2→3), not impact ranking
3. **Test strategy:** Implement all changes first, then batch-fix broken tests + add new integration tests
4. **Tier architecture:** Remove Tier 1/2 entirely; adopt stochastic activation where all activated agents use LLM
5. **Bayesian belief:** Implement true Bayes formula (not just rename)
6. **Rationale for full LLM:** Background agents with LLM can produce unexpected emergent behavior that rule-based agents cannot — this is the engine's core value proposition

### Spec Review Corrections (rev 2)

- **C1 fixed:** Bayesian formula now accounts for dual belief scales (HK [-1,+1] stance vs kg_driven [0,1] probability)
- **C2 fixed:** Debate writeback already exists in-memory; real gap is DB persistence only
- **C3 fixed:** `is_stakeholder` infrastructure fully scoped as net-new work
- **I1-I7 fixed:** All file references corrected to actual function/file names
- **S2 adopted:** `response_delay` deferred (not added to schema)
- **S4 adopted:** `KGSessionState.tier1_agents` renamed to `stakeholder_agents`

## Phase 0 — Tier Removal + Stochastic Activation

### Current State

```
simulation_runner.py:
  → _load_kg_driven_state() lines ~1387-1407: SELECT TOP 30 by importance → stored in kg_state.tier1_agents
  → Group 2: kg_driven hooks guard with `if agent_id in kg_state.tier1_agents`
  → agent_profiles.tier column: added via ALTER TABLE at runtime (INTEGER DEFAULT 2)
```

### New Architecture

**Stochastic Activation Model:**

1. **Agent Profile** gains 2 behavioral params (generated by KGAgentFactory via LLM, or inferred from demographics in HK mode):
   - `activity_level: float [0,1]` — per-round activation probability
   - `influence_weight: float [0,3]` — post visibility weight in feed ranking
   - ~~`response_delay`~~ — deferred to future work

2. **`is_stakeholder` — net-new infrastructure:**
   - New column: `agent_profiles.is_stakeholder BOOLEAN DEFAULT 0`
   - New env var: `AGENT_LLM_MODEL_LITE` (default: same as `AGENT_LLM_MODEL`)
   - `scenario_generator.py`: LLM prompt extended to output `stakeholder_entity_types: list[str]` alongside existing decision_types/metrics/shocks
   - `kg_agent_factory.py`: after profile creation, match agent's entity_type against stakeholder_entity_types → set `is_stakeholder=True`
   - HK mode (`agent_factory.py`): infer from occupation (政府官員/議員/地產商/銀行家 → stakeholder)

3. **Per-round activation** via `get_active_agents_for_round()`:
   - Each agent independently: `random() < activity_level`
   - All active agents → LLM deliberation (no tier distinction)
   - Inactive agents → silent this round (retain previous state)

4. **Model routing** (cost control):
   - `is_stakeholder=True` agents → `AGENT_LLM_MODEL` (stronger model)
   - `is_stakeholder=False` agents → `AGENT_LLM_MODEL_LITE` (flash-lite)
   - Both paths are real LLM — quality differs, not mechanism
   - `llm_client.py`: new `get_agent_model(is_stakeholder: bool) → tuple[str, str]` returning (provider, model)

5. **Semaphore:** `max_concurrent = min(50, active_count)`

6. **lite_hooks.py** new scope:
   - No longer used in main simulation loop
   - Serves SwarmEnsemble Phase B replicas only

7. **Stakeholder floor:** `is_stakeholder=True` → `activity_level = max(activity_level, 0.8)`

8. **Reproducibility:** New `activation_seed` field on simulation session request model (not preset); fixed seed → deterministic activation sequence via `random.Random(seed)`

9. **KGSessionState refactoring:** Rename `tier1_agents: list[str]` → `stakeholder_agents: list[str]`; remove all tier-related comments

### HK Mode activity_level Inference Rules

```
Age 18-35:  0.7-0.9
Age 36-55:  0.4-0.6
Age 55+:    0.2-0.4
High income: influence_weight × 1.3
Civil servant/teacher/journalist: activity_level + 0.1
```

### Cost Estimate (STANDARD: 300 agents, 20 rounds)

```
Average activity_level ≈ 0.5, stakeholder floor 0.8 → ~160 active agents/round
Stakeholders ~30 → gemini-flash ($0.002/call)
Background ~130 → gemini-flash-lite ($0.0003/call)
Per-round cost: 30×0.002 + 130×0.0003 = $0.099
20 rounds: $1.98 (vs current ~$1.89 — ~5% increase)
```

## Phase 2 — Core Wiring (Group 2)

### ④ Feed Ranking → Agent Cognition Loop

**Gap:** `agent_feeds` table populated but agents never read their feed during decisions.
**Fix:** Each active agent retrieves top-5 feed items as LLM decision context.

```
Flow:
  Group 2, before decision:
    feed_items = await feed_ranker.get_agent_feed(agent_id, round, limit=5)
    → inject into LLM prompt as {feed_context}
    "你最近睇到嘅帖文：\n{feed_items_formatted}"
```

**Anti-echo-chamber:** Feed ranking adds diversity injection — at least 1/5 items from opposing stance.

**Files:** `simulation_runner.py`, `backend/prompts/decision_prompts.py` + `cognitive_agent_engine.py` (add `{feed_context}`), `feed_ranker.py` (add `get_agent_feed()` query helper)

### ⑤ Bayesian Belief Update (True Implementation)

**Gap:** Current system uses linear weighting (Anderson Integration Theory), mislabeled as "Bayesian."
**Fix:** Implement true Bayes formula, handling BOTH belief scales.

**CRITICAL: Dual belief scale handling:**

- **kg_driven mode** uses `[0, 1]` probability scale (neutral = 0.5). Standard Bayes applies directly:
  ```
  prior = belief[topic]                              # [0, 1]
  lr = compute_likelihood_ratio(event, agent)        # P(E|H) / P(E|¬H)
  prior_odds = prior / (1 - prior)
  posterior_odds = prior_odds × lr
  posterior = posterior_odds / (1 + posterior_odds)
  clamp to [0.02, 0.98]
  ```

- **HK mode** uses `[-1, +1]` stance scale (neutral = 0). Transform to probability domain first:
  ```
  stance = belief.stance                             # [-1, +1]
  prob = (stance + 1) / 2                            # map to [0, 1]
  lr = compute_likelihood_ratio(event, agent)
  prob_posterior = bayesian_update(prob, lr)          # same formula as above
  stance_posterior = prob_posterior * 2 - 1           # map back to [-1, +1]
  clamp to [-0.98, 0.98]
  ```

**Likelihood ratio computation** (shared by both modes):
```
base_lr = 1 + abs(event.impact) × event.credibility    # evidence strength
if event aligns with agent's current belief direction:
    lr = base_lr × (1 + confirmation_bias × 0.3)       # confirmation boost
else:
    lr = 1 / (base_lr × (1 - confirmation_bias × 0.3)) # contradicting evidence
```

**Files:**
- `belief_system.py` (new `bayesian_update()` + `_stance_to_prob()` + `_prob_to_stance()` + `compute_likelihood_ratio()`)
- `belief_propagation.py` (cascade uses `bayesian_update` on [0,1] scale — no transform needed)

### ⑥ Debate Delta → DB Persistence

**Finding (spec review):** Debate writeback to `kg_state.agent_beliefs` (in-memory) already exists at simulation_runner.py lines ~1891-1905. The actual gap is that updated beliefs are **not persisted to `belief_states` DB table** after debate.

**Fix:** After debate in-memory writeback, persist updated beliefs to DB.

```
Flow:
  After existing in-memory writeback (lines ~1891-1905):
    for agent_id, topic_deltas in deltas.items():
        for topic, delta in topic_deltas.items():
            new_value = kg_state.agent_beliefs[agent_id][topic]
            await db.execute(
                "INSERT OR REPLACE INTO belief_states (session_id, agent_id, topic, stance, round_number) VALUES (?,?,?,?,?)",
                (session_id, agent_id, topic, new_value, round_num)
            )
```

**Files:** `simulation_runner.py` (add DB persist after existing in-memory writeback, ~5 lines)

## Risk Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Stochastic activation increases run variance | Hard to reproduce bugs | `activation_seed` parameter; fixed seed → deterministic activation |
| Full LLM increases per-round latency | STANDARD ~30s/round → ~60s/round | Semaphore + batch requests + async gather |
| Bayesian update unstable near extremes | NaN or belief lock | Clamp [0.02, 0.98] (prob) / [-0.98, 0.98] (stance); unit tests cover edge cases |
| HK↔prob scale transform introduces rounding drift | Belief values drift over many rounds | Round to 4 decimal places after each transform cycle |
| Feed→cognition creates echo chamber positive feedback | Beliefs converge too fast | Feed diversity injection: ≥1/5 posts from opposing stance |
| Branch deep copy increases DB I/O | Branch creation slower | `INSERT INTO ... SELECT` batch copy |
| Removing tier guards breaks unknown dependencies | Unexpected test failures | Run full test suite after Phase 0 to establish baseline |
| Activity_level inference inaccurate | Key agents silenced | Stakeholder `activity_level` floor = 0.8 |
| Schema migration is one-way | SQLite cannot easily DROP COLUMN | Accept as forward-only migration; new columns have safe defaults |
